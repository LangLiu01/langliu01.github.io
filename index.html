Git笔记:挖坑指南一（安装）

2018/11/1

这是我在学习Git的时候作的一些笔记，好记性不如烂笔头，所以我把它写这这里，以便将来复习的时候作参考

题外话:我实在是太喜欢Lady Gaga 的那张专辑Joanne了，你甚至可以在下面的那堆小卡片里找到这张专辑的图片...好啦不说了，进入正题

版本控制系统VCS:

版本控制系统的简洁定义：一个可以用于管理和追踪软件代码或者其他内容的不同版本的工具。它常常用于开发及维护代码，读取代码的不同版本 并记录所有的修改记录，这样我们就可以知道是哪个倒霉的程序员在版本提交里引入了BUG了，嗯，真好。

Git是一种适用于分布式、多人协同开发的版本控制系统。简介到此为止。要想知道更多关于Git是如何诞生的历史故事，访问git-scm的官方 网址，然后，慢慢看吧。(注：git-scm由于在国内不明原因，访问常常不稳定，要多试几次。)

安装

在这篇博客里，我主要会谈到linux下尤其是Ubuntu下的安装，为什么？因为我用的是Ubuntu,而且这又是我的博客，记的又是我的笔记 所以，没有那么多为什么。

方法1： 编译二进制发行版

最困难的安装方法，适合那些对linux系统环境非常熟悉的大佬使用，对我这种小白来说，还是算啦吧，我可不想在编译出错的时候对一大堆 看不懂的环境变量报错信息发呆。直接跳过此方法 ：）

方法2：用Ubuntu的包管理器轻松安装

首先要添加依赖关系:

sudo add-apt-repository ppa:git-core/ppa apt update apt install git

如果系统自带了Git要把它升级成最新版本：

git clone https://github.com/git/git

但是经我自己试验，这个方法十之八九是行不通的，受限于糟糕的网络环境，clone过程中经常连接中断，很是烦人。要是有免费又稳定的VPN就好了

配置Git

首先要设置用户信息，这样每次提交的时候就不用重复输入这些信息了

git config --global user.name "Lang Liu" git config --global user.email "liulang1101@163.com"

然后还可以配置linux下你最喜欢的文本编辑器，通常是Vim/Vi/Emacs，但是我还不会用这些牛逼的编辑器，所以下面这条命令也就没有试过

git config--global core.editor emacs/vim

最后，我们来检查一下配置信息，看看还有没有要修改的

git config --list

帮助

谁都有需要帮助的时候，下面的这些命令可以让你查看用户手册

git help

是指某一条具体的命令,还有别忘了这是linus Towards开发的linux系统，你可以随时用man来查看任意一个命令的帮助手册

man git-

写在最后：要是嫌弃这些死板的用户手册或者是遇到某些特定问题了，这种时候要请教比我们强的多的人。github 上面的issue可以问， stackflow 可以问，linux.com也可以问，要是实在不行，还有百度知道这种平台可以请教问题，这个世界上还是有好多热心人的。别怕， 尽管问。

Git笔记：挖坑指南2（基本命令）

2018/11/2

今天要写的是关于git的基本命令，但是有一个部分我可能没法把它写好，关于文件在不同情况下的状态，所以今天的博客又挖了一个坑， 还是要小心谨慎为好。

TIPS：关于文件状态的知识你还是需要访问git-scm，并且阅读Pro Git Book (ebook)PDF版,有汉化。那本书讲解十分详细。

初始化仓库(repository)

首先选中现有的工作目录或者空目录

git init

然后如果是选择了工作目录，想要把所有文件添加到git，

git add*

然后添加LINCENSE文件，授权协议历史非常长而且很复杂，详细内容请自行百度，

git add LINCENSE

如果在添加过程中，一些程序自动生成的“副产品”不想添加到git里面，创建一个.gitignore文件

cat .gitignore

在文件中添加*.[txt] 那么以txt结尾的文件在添加时会被pass掉。

Git clone克隆/复制

git clone [url]

检查文件状态status(坑)

经常检查文件的状态信息，可以避免无法提交或者提交失败的尴尬

git status

git status-s可以输出更为简洁的信息

在git 中文件一共有三种不同的状态可能性：未添加、已添加在暂存区、提交区，要经常检查文件的状态信息确保全部提交

总结:基本流程

添加文件 git add

提交到暂存区git commit -m "beta version"后面输入改动信息，不输入改动信息无法正确提交

推送到远程仓库 git push orign master
